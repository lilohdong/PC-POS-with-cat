# 프로그램 상세 

## 프로그램 개요 

### 매장관리
          매장의 좌석관련 업무를 총괄하여 관리할 수 있는 페이지.
          실시간으로 좌석 정보 업데이트하여 정보 표시하는 기능
          좌석의 사용 시작, 종료 기능
          시간 충전 기능
          좌석 이용 정보 조회 기능(이용중인 회원 및 시간 관련 정보)
          고장 등의 이유로 이용 불가능한 좌석 설정 기능
          좌석 이용 정보 통계 표시
          시스템 시간 표시
          정전, 시스템 다운 등을 대비한 실시간 DB 상태 정보 연동 기능

### 상품판매
          상품 판매 화면 및 주문하기 화면은 손님의 메뉴 주문을 실시간으로 접수하고
          처리하는 화면.          사용자가 카테고리(라면, 볶음밥 등) 버튼을 클릭하거나 검색창에 메뉴를 입력하면
          해당 메뉴 목록을 동적으로 표시.          메뉴를 클릭하여 장바구니에 추가, 수량 증가/감소, 제거할 수 있고, 재고가 부족한
          메뉴는 주문에 추가 불가능.          좌석 번호(1 ~ 100 검증), 결제 수단, 요청사항을 입력하여 주문 접수 시 DB에
          저장하고, 매출을 자동으로 변경 및 재료들을 자동 차감.          조리중/완료 탭 전환으로 주문 상태를 나누어 볼 수 있고, 준비완료/취소 버튼, 상태
          팝업으로 주문 상세내역 조회 가능.

### 재고관리
          재고 목록이 테이블 형식으로 나타나 각 재료의 코드, 이름, 카테고리, 현재/최소 재고,
          위치 확인 가능.          상품명/코드/카테고리 검색, 상태 필터(정상/부족/품절)를 통하여 원하는 재료 확인 가능.          테이블의 재료를 선택하고 입고/출고 버튼을 눌러 개수를 수동적으로 변경 가능
          (재고 부족 시 출고 불가).          하단 테이블에서 실시간으로 입고(초록), 출고(주황), 부족(빨강) 로그 확인 가능.          (현재 재고 < 최소 재고)인 재료의 경우 하단의 테이블로 자동 표시되며, 알림 버튼으로
          필터링하여 확인 가능.

### 회원관리
          PC방 이용 고객의 회원 정보를 효율적으로 관리하기 위한 기능임. 
          회원은 가입을 통해 이름, 연락처 등의 기본 정보를 입력하여 시스템에 등록되며,
          등록된 회원 정보는 데이터베이스의 회원 테이블에 저장됨.          관리자는 특정 회원을 선택해 정보 수정이 가능하며, 회원 탈퇴 요청 시 해당 회원을
          삭제하여 관리할 수 있음.           이 기능을 통해 회원 가입, 조회, 수정, 삭제가 체계적으로 이루어질 수 있음.

### 인수인계
          직원 간 근무 시 매출,현금,근무 시간 정보를 정확히 전달하기 위한 기능임.          근무를 인계하는 직원은 로그인 후 근무 종료 시점의 매출 데이터를 확인할 수 있으며,
          판매된 상품 금액과 시간 충전 금액 등이 자동 집계됨.          특히 현금 매출의 경우 시스템에 기록된 금고 금액과 실제 현금 금액을 비교하여 업무
          차액이 자동 계산되어 표시됨.          근무를 인수하는 직원은 본인의 정보로 로그인하여 인수 절차를 진행하며, 이를 통해
          정확한 교대 업무 처리가 가능.

### 매출 관리
          PC방의 총 매출을 확인할 수 있는 기능
          음식 판매 매출 부분과 시간 상품 판매 매출 부분을 나눠서 볼 수 있으며,
          정상적으로 판매 완료되어 매출이 발생한 부분을 전부 집계
          조회 기능을 통해 일간, 주간, 월간 단위로 매출을 볼 수 있으며 일간의 경우 시간대에
          따라 매출 발생 여부 확인 가능

### 직원 관리
          PC방 직원들의 정보 조회/수정, 입사/퇴사등의 업무를 처리하는 기능
          재직중인 직원들의 목록을 검색하는 기능

### 게임 통계
          PC방의 총 매출을 확인할 수 있는 기능
          음식 판매 매출 부분과 시간 상품 판매 매출 부분을 나눠서 볼 수 있으며, 정상적으로
          판매 완료되어 매출이 발생한 부분을 전부 집계
          조회 기능을 통해 일간, 주간, 월간 단위로 매출을 볼 수 있으며 일간의 경우 시간대에
          따라 매출 발생 여부 확인 가능


## 프로그램의 기능 및 특성 설명
### 사이드바
- 관리자 모드에 진입해야만 매출관리와 직원 관리 기능에 접근할 수 있게 하여, 일반 직원이 민감한 부분에 접근하는 것을 막음.
- 다크 모드 – 라이트 모드 전환 기능을 추가하여 사용자 편의성을 높임.
- 일반적인 탭 기반이 아닌 네비게이션 구조를 채택하여 직관성을 높임.

### 매장관리
- 좌석을 Grid 형태로 배치하여 직관적으로 좌석을 관리 가능하게 설계하였음.
- 정전, 프로그램 다운 등에 대비하여 현재 이용중인 좌석들의 상태 정보를 주기적으로 DB에 업데이트하여 추후 복구 시 PC방 상태 정보를 최신으로 유지하도록 구현함.
- 좌석 이용 정보를 수정(시작, 종료 등)시 확인 단계를 거치게 하여 안정성을 확보함.

### 상품판매 
- 주문을 취소하거나 환불할 시 트랜잭션을 통해 매출이 자동으로 차감됨
- 매출 데이터의 정확성을 유지할 수 있는 장점이 있음.
- 환불 로직은 구현하였으나, 재고 복구는 PC방 근무환경과 음식 물 판매 수칙을 고려하여 구현하지 않음 - 필요에 따라 추후에 구현 예정.

### 재고관리
- 담당한 파트들의 주요 장점은 자동화와 실시간 연동으로 인한 운영 효율성 향상.
- 여러 가지 기능을 도입해 수동 입력 오류를 최소화하고, 재고 데이터를 기반으로 한 동적 처리를 통해 손실을 방지함.
- 주문 접수 시 메뉴 레시피(menu_ingredient 테이블)를 기반으로 필요한 재료를 자동계산해 재고- 를 차감.
    - 이는 매장 직원이 수동으로 재고를 업데이트할 필요가 없어 오류가 줄고, 재고 부족 시에 자동 알림- 이 발생함. 기존 POS시스템과 다르게 이 시스템은 재고 단위수 -> 재료
          단일개수로 변환하는 로직- 을 도입한 뒤 주문 시 재료 소진을 실시간으로 반영함.
- 동시 주문 시 재고 초과를 방지하기 위해 재고 관련 롤백을 강화함.
- 재고 입/출고 시 박스 단위로 입력하면 stock_info.unit_quantity를 통해 단일 개수로 변환하여 재고를 업데이트함.
- 입력 오류를 줄이고, 실제 재고 관리를 쉽게 할 수 있음.
- 트리거를 활용해 DB 수준에서 자동 변환하는 점이 다른 POS시스템과의 독창적인 요소.

### 회원관리
- 중복 회원 방지를 위해 ID중복검사 기능을 추가 했음.
- 회원 정보 수정 시 실시간으로 반영되어 표기 되도록 했음.
- 회원 삭제시 안정성을 위해 확인하는 팝업을 표기했음.

### 인수인계
- 현금으로 판매가 된 경우에 인계자와 인수자가 각각 현금을 확인할 때 직접 개수를 세어 계산해야한다는 번거로움을 덜어내기 위해 화폐 종류별로 개수만 적어 넣으면 합산한 값을 자동으로 도출해주는 자동 계산 기능을 넣었음. 

### 매출 관리
- 매출 관리 부분을 시간 판매 내역과 상품 판매 내역으로 분리하여 PC방 매출이 어떤 요소에 더 크게 의존하는지 직관적으로 파악할 수 있게 함
- 테이블 내부에 주문 번호, 회원 정보, 메뉴 명, 수량, 금액 등 다양한 정보를 고밀도로 배치하여 한 화면에서 전체 매출 흐름을 빠르게 파악할 수 있게 함

### 직원 관리
- 퇴사한 직원에 대한 기록을 유지하기 위하여 직원 테이블에 현재 재직중인지 여부를 저장하는 속성을 추가하였음.
